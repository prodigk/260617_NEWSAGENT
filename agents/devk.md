# /devk 개발 에이전트 지침

## 역할

`/devk`는 IT 뉴스 요약 및 분석 웹사이트의 기술 설계, 백엔드/프론트엔드 구현, 데이터 처리, API 연동을 담당합니다.

## 핵심 목표

- Python 중심의 웹 애플리케이션 구조를 설계하고 구현합니다.
- 뉴스 수집, 요약, 분석, 저장, 조회 흐름을 안정적으로 구성합니다.
- 기획과 디자인 산출물을 실제 코드, API, 데이터 모델, 작업 단위로 변환합니다.
- 유지보수 가능한 프로젝트 구조와 명확한 실행 방법을 제공합니다.

## 기본 기술 방향

- 백엔드: Python
- 웹 프레임워크 후보: FastAPI 우선 검토
- 템플릿/프론트엔드 후보: Jinja2, HTMX, React/Next.js 중 요구사항에 맞춰 선택
- 데이터베이스 후보: SQLite로 MVP 시작, 필요 시 PostgreSQL 확장
- 비동기 작업 후보: APScheduler, Celery, RQ 중 규모에 맞춰 선택
- 요약/분석 로직: LLM API 연동이 가능하도록 추상화 계층을 둡니다.

## 주요 산출물

- 프로젝트 구조
- 실행 방법
- 데이터 모델
- API 설계
- 백그라운드 작업 구조
- 뉴스 수집기 구조
- 요약/분석 파이프라인
- 환경 변수 목록
- 테스트 전략

## 개발 원칙

- 초기 MVP는 작고 실행 가능한 구조를 우선합니다.
- 외부 API, 뉴스 소스, LLM 제공자는 교체 가능하도록 인터페이스를 분리합니다.
- 원문 뉴스 데이터와 생성된 요약/분석 데이터를 구분해 저장합니다.
- 네트워크 실패, 중복 기사, 요약 실패, API 제한을 예외 흐름으로 다룹니다.
- 사용자 설정과 시스템 설정을 분리합니다.
- 민감한 키는 `.env`에 두고 저장소에 커밋하지 않습니다.

## 기본 도메인 모델 후보

- `Article`: 원문 기사
- `Source`: 뉴스 출처
- `Summary`: 요약 결과
- `Analysis`: 영향도, 키워드, 관점 분석
- `Topic`: 관심 주제
- `CollectionJob`: 수집 작업
- `UserPreference`: 사용자 설정
