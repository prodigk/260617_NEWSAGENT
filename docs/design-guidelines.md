# IT 뉴스 분석 사이트 디자인 지침

## 목적

이 문서는 `docs/site-plan.md`의 기능을 실제 화면으로 설계하기 위한 디자인 지침입니다. 첨부 이미지 `newssite_design.png`의 톤앤매너를 바탕으로 하되, 단순 뉴스 매거진이 아니라 최신뉴스 수집, 기사 요약, 키워드 분석, 감정 분석, 관련기사 추천을 수행하는 분석형 웹사이트에 맞게 조정합니다.

## 디자인 브리프

- 제품: 한국어 중심 IT 뉴스 요약 및 분석 웹사이트
- 주요 사용자: 개발자, 기획자, IT 실무자, 기술 트렌드 추적자
- 주요 행동: 최신기사 불러오기, 뉴스 탐색, 요약 확인, 키워드/감정 분석 확인, 관련기사 탐색, 의미 검색
- 시각 기준: 크림색 지면 위에 굵은 세리프 헤드라인, 흑백 이미지, 얇은 구분선, 하이라이트 텍스트가 있는 에디토리얼 뉴스 레이아웃
- 인터랙션 수준: MVP에서는 핵심 필터, 검색, 상세 진입, 수동 갱신, 추천 클릭이 실제로 동작해야 함
- 뉴스 탐색 방식: 첨부 이미지처럼 한 페이지에 여러 기사가 배치되는 에디토리얼 그리드를 기본으로 하고, 하단 페이지네이션으로 다음 그리드 페이지를 전환

## 첨부 이미지에서 추출한 톤앤매너

### 전체 인상

- 신문 1면과 매거진 피드를 섞은 편집형 레이아웃
- 넓은 여백과 명확한 세로 구분선으로 정보 덩어리를 나눔
- 강한 장식보다 타이포그래피와 이미지 대비로 분위기를 만듦
- 디지털 앱이라기보다 신뢰감 있는 편집실, 리서치 노트, 브리핑 문서에 가까움

### 레이아웃 특징

- 상단에는 낮고 넓은 헤더
- 중앙에는 섹션 제목과 짧은 밑줄
- 본문은 3-4열 신문식 그리드
- 주요 기사는 더 큰 폭과 큰 제목으로 강조
- 보조 기사는 좁은 칼럼에 축약 배치
- 카드 박스보다 얇은 선과 여백으로 영역을 구분

### 타이포그래피 특징

- 제목은 굵고 대비가 큰 세리프 계열
- 보조 텍스트와 UI 텍스트는 단정한 산세리프 계열
- 제목은 줄바꿈을 적극 활용해 리듬을 만듦
- 중요한 단어는 배경 하이라이트로 강조
- 본문은 넉넉한 행간으로 읽기 쉽게 구성

### 색상 특징

- 배경은 순백색이 아니라 따뜻한 크림색
- 텍스트는 거의 검정에 가까운 잉크색
- 이미지와 미디어는 흑백 또는 낮은 채도
- 강조 색은 소량의 피치, 옐로우, 민트/시안 계열 하이라이트
- CTA는 검정 배경과 크림색 텍스트로 강하게 처리

### 컴포넌트 특징

- 버튼은 각진 직사각형에 가까움
- 둥근 모서리는 작게 사용
- 프로필, 상태 아이콘처럼 작은 요소에만 둥근 형태 허용
- 하이라이트는 배지보다 텍스트 뒤에 칠한 형광펜처럼 보이게 처리
- 구분선은 매우 얇고 밝은 회색/베이지 톤

## 디자인 파운데이션

### 색상 토큰

```text
surface.paper: #F5F0E8
surface.panel: #FBF8F1
surface.raised: #FFFDF8
ink.primary: #111111
ink.secondary: #3E3A35
ink.muted: #7A736A
line.subtle: #DDD4C8
line.strong: #111111
accent.peach: #FFB59D
accent.yellow: #F6E36D
accent.mint: #C8E8E8
accent.positive: #2E7D50
accent.neutral: #7C7166
accent.negative: #A13D3D
```

### 타이포그래피

- 헤드라인: 세리프 계열. 후보는 `Noto Serif KR`, `Source Serif 4`, `Georgia`.
- UI/본문: 산세리프 계열. 후보는 `Pretendard`, `Inter`, `Noto Sans KR`.
- 대시보드 H1: 40-56px, 굵게, 줄간격 1.05-1.15
- 섹션 제목: 22-30px, 세리프 또는 굵은 산세리프
- 뉴스 제목: 24-38px, 세리프
- 카드/리스트 제목: 18-22px, 세리프
- 본문/요약: 15-17px, 줄간격 1.55-1.75
- 메타/상태: 12-14px, 산세리프

### 간격과 그리드

- 데스크톱 최대 콘텐츠 폭: 1720px
- 기본 페이지 패딩: 40-64px
- 모바일 페이지 패딩: 20px
- 데스크톱 기본 그리드: 12컬럼
- 대시보드 추천 레이아웃: 좌측 브리핑 3컬럼, 중앙 메인뉴스 4컬럼, 우측 분석/추천 5컬럼
- 뉴스 목록 추천 레이아웃: 4개 세로 칼럼을 기본으로 하되 대표 기사는 2칼럼 폭을 차지할 수 있음
- 칼럼 사이에는 카드 배경보다 얇은 세로 구분선을 우선 사용
- 반복 리스트는 1px 구분선과 충분한 상하 패딩으로 나눔

### 이미지 방향

- 뉴스 썸네일은 가능하면 흑백 또는 낮은 채도 필터를 적용
- 이미지가 없을 때는 빈 박스 대신 제목, 키워드, 출처 중심의 타이포그래피 블록 사용
- 분석 화면에서는 이미지보다 데이터와 요약을 우선
- 썸네일 비율은 `16:10`, `4:3`, `1:1` 중 화면 맥락에 맞춰 고정

### 하이라이트 규칙

- 키워드 하이라이트: `accent.mint`
- 중요한 시장/산업 변화: `accent.yellow`
- 경고성 이슈 또는 부정 감정: 옅은 피치/레드 계열
- 하이라이트는 텍스트 뒤 배경으로 사용하고, 전체 문장보다 핵심 단어 1-4개에만 적용
- 감정 분석은 색상만으로 표현하지 않고 `긍정`, `중립`, `부정` 텍스트를 함께 표시

## 사이트 구조 디자인

### 1. 글로벌 헤더

역할:

- 브랜드 인지
- 주요 화면 이동
- 수동 뉴스 갱신
- 수집 상태 확인

구성:

- 좌측: 서비스 로고/이름 `IT Brief`
- 중앙: 대시보드, 뉴스, 키워드, 감정, 의미검색, 수집상태
- 우측: 마지막 수집 시간, `최근기사 불러오기` 버튼, 메뉴 버튼

디자인:

- 높이 80-96px
- 배경은 `surface.paper`
- 하단 구분선은 사용하지 않거나 매우 옅게 사용
- `최근기사 불러오기`는 검정 배경의 주요 CTA
- 모바일에서는 로고, 갱신 버튼, 메뉴 아이콘만 노출

### 2. 대시보드

목표:

- 사용자가 오늘의 IT 흐름을 1분 안에 파악하게 함

레이아웃:

- 상단 중앙 섹션 제목: `Latest IT Brief`
- 제목 아래 짧은 검정 밑줄
- 본문은 에디토리얼 3영역 구조

영역:

- 좌측 브리핑 칼럼: 수집 상태, 전체 기사 수, 분석 완료율, 감정 분포
- 중앙 메인 칼럼: 가장 중요한 기사 1개와 주요 요약
- 우측 인사이트 칼럼: 상위 키워드, 급상승 이슈, 추천 기사 묶음

필수 UI:

- `최근기사 불러오기` 버튼
- 수집 중 로딩 상태
- 수집 실패 상태
- 분석 대기/완료 카운트
- 상위 키워드 하이라이트
- 긍정/중립/부정 분포

사용성 지침:

- 수집/분석 상태는 사용자가 기다릴지 판단할 수 있게 숫자로 보여줌
- 메인 기사는 제목보다 먼저 카테고리, 출처, 발행 시간을 작게 표시
- 요약은 최대 3줄까지만 노출하고 상세 화면으로 연결

### 3. 뉴스 목록

목표:

- 최대 50개 기사를 페이지 단위의 에디토리얼 그리드로 훑고 필터링하게 함

레이아웃:

- 상단: 검색, 필터, 정렬, 보기 전환
- 중앙: 에디토리얼 그리드
- 하단: 페이지네이션 `1 2 3 4 5 ...`
- 보조 영역: 선택한 키워드/감정 요약 또는 추천 묶음

그리드 슬롯 구성:

- 대표 기사 슬롯: 큰 제목, 큰 이미지, 3줄 요약, 출처 프로필, 키워드, 감정
- 중간 기사 슬롯: 이미지, 제목, 한 줄 요약, 출처, 키워드
- 짧은 기사 슬롯: 제목, 출처, 발행일, 키워드
- 분석 슬롯: 상위 키워드, 감정 분포, 관련기사 묶음
- 추천 슬롯: 기준 기사와 관련기사 2-3개

각 슬롯 공통 구성:

- 기사 제목
- 기사 이미지 또는 타이포그래피 대체 블록
- 한 줄 요약 또는 본문 일부
- 출처명
- 출처 프로필 이미지 또는 로고
- 발행일
- 핵심 키워드
- 감정 라벨과 점수
- 관련기사 수
- 분석 상태

페이지네이션:

- 그리드 하단 중앙에 배치
- 현재 페이지는 검정 배경 또는 진한 밑줄로 표시
- 숫자 사이 간격은 넉넉하게 두어 신문 면 번호처럼 보이게 함
- `이전`, `다음`은 텍스트보다 화살표 아이콘을 우선
- 페이지 전환 시 스크롤 위치는 그리드 상단으로 이동
- 모바일에서는 `1 2 3 ...` 대신 현재 페이지와 이전/다음 중심으로 간소화 가능

사용성 지침:

- 필터는 카테고리, 키워드, 감정, 출처, 정렬 순으로 배치
- 필터 적용 결과 수를 즉시 표시
- 기사 제목 클릭 시 상세로 이동
- 키워드 클릭 시 해당 키워드 필터 적용
- 감정 라벨 클릭 시 해당 감정 필터 적용
- 페이지네이션 이동 시 필터와 검색 조건은 유지
- 필터 결과가 적으면 그리드 빈 슬롯을 만들지 않고 남은 기사를 재흐름 처리

### 4. 뉴스 상세

목표:

- 하나의 기사가 왜 중요한지와 관련 맥락을 이해하게 함

레이아웃:

- 상단: 제목, 출처, 발행일, 원문 링크
- 본문 좌측: 요약과 분석
- 본문 우측: 키워드, 감정 분석, 관련기사 추천
- 하단: 원문 본문 일부 또는 추출 상태

핵심 섹션:

- 한 줄 핵심 요약
- 3줄 상세 요약
- 왜 중요한가
- 더 확인할 포인트
- 핵심 키워드
- 감정 분석
- 관련기사 추천
- 추천 이유

사용성 지침:

- 요약과 분석을 원문보다 먼저 보여줌
- 원문 링크는 항상 보이는 위치에 둠
- 관련기사 추천에는 `추천 이유`와 `공통 키워드`를 함께 표시
- 본문 추출 실패 시 실패 사실과 대체 분석 기준을 명확히 표시

### 5. 키워드 분석

목표:

- 수집된 50개 기사에서 가장 중요한 기술 이슈를 발견하게 함

레이아웃:

- 상단: 상위 키워드 하이라이트 스트립
- 본문: 키워드별 기사 수와 감정 분포
- 우측 또는 하단: 선택 키워드 관련 기사 목록

디자인:

- 키워드는 형광펜 하이라이트와 숫자를 함께 사용
- 순위는 과하게 장식하지 않고 타이포그래피 크기 차이로 표현
- 키워드 클릭 시 관련 기사 묶음이 갱신됨

### 6. 감정 분석

목표:

- IT 뉴스 흐름의 분위기를 파악하게 함

레이아웃:

- 상단: 긍정/중립/부정 전체 비율
- 본문: 카테고리별 감정 분포
- 하단: 부정 이슈와 긍정 이슈 대표 기사

디자인:

- 감정 분포는 막대, 작은 도트, 텍스트 비율로 표현
- 빨강/초록을 과하게 쓰지 않고 잉크색 기반에 보조색만 사용
- 감정 분석이 사실 판단이 아니라 기사 논조/시장 반응 해석임을 작게 안내

### 7. 의미 검색

목표:

- 사용자가 자연어로 이슈를 입력해 관련 기사를 찾게 함

레이아웃:

- 상단: 큰 검색 입력
- 본문: 관련도 높은 기사 리스트
- 우측: 검색 결과의 키워드와 감정 분포

사용성 지침:

- 검색 예시를 입력창 아래에 배치
- 결과에는 관련도 점수 또는 `높음/보통/낮음`을 표시
- 결과가 없을 때는 검색어 수정 제안과 인기 키워드를 보여줌

### 8. 수집 상태

목표:

- News API, 본문 추출, LangChain/OpenAI 분석, Chroma 저장 상태를 확인하게 함

레이아웃:

- 상단: 마지막 작업 요약
- 본문: 단계별 상태 타임라인
- 하단: 실패 기사 목록과 사유

상태 단계:

- News API 수집
- 중복 제거
- 원문 본문 추출
- 요약 생성
- 키워드 분석
- 감정 분석
- 임베딩 생성
- Chroma 저장
- 관련기사 추천 생성

- 신문 칼럼처럼 얇은 구분선 안에 배치
- 큰 제목, 짧은 요약, 메타, 키워드, 감정 라벨 순서
- 이미지가 있으면 흑백 썸네일을 상단 또는 좌측에 고정 비율로 배치
- 출처 프로필 이미지 또는 로고를 작은 원형/각진 썸네일로 표시
- 출처 이미지가 없으면 출처명의 첫 글자를 사용한 단정한 텍스트 마크로 대체

### Editorial Grid

- 첨부 이미지의 4칼럼 신문식 구성을 한 페이지의 기본 레이아웃으로 사용
- 페이지당 8-12개 기사 배치
- 대표 기사 1개는 큰 이미지와 큰 세리프 제목으로 강조
- 이미지가 있는 기사와 이미지가 없는 기사를 섞어 지면 리듬을 만듦
- 각 칼럼 사이에는 얇은 세로선을 사용
- 그리드 안에서는 둥근 카드 배경을 피하고, 지면 위에 직접 편집된 느낌을 유지

### Editorial Slot

- `hero`: 페이지 대표 기사, 가장 큰 제목과 이미지
- `feature`: 중간 크기 기사, 이미지와 요약 포함
- `brief`: 짧은 기사, 이미지 없이 제목과 메타 중심
- `related`: 관련기사 추천 묶음

### Pagination

- 숫자 페이지네이션은 지면 하단의 페이지 번호처럼 보이게 설계
- 현재 페이지는 검정 잉크색으로 강하게 표시
- 비활성 숫자는 `ink.muted`
- 키보드 포커스와 hover 상태를 명확히 제공
- 페이지 변경 중에는 그리드 영역에 짧은 로딩 상태를 표시

### Highlight Keyword

- 텍스트 뒤에 형광펜처럼 배경색 적용
- 모서리는 0-2px
- 패딩은 좌우 2-4px 수준

### Sentiment Label

- 라벨 텍스트와 점수 동시 표시
- 긍정/중립/부정 색상은 작은 선, 점, 배경 틴트에만 사용

### Refresh Button

- 검정 배경, 크림색 텍스트
- 로딩 중에는 비활성화하고 진행 상태 표시
- 실패 시 버튼 옆에 짧은 오류 메시지 표시

### Analysis Strip

- 기사 카드 하단에 `요약`, `키워드`, `감정`, `관련기사` 상태를 한 줄로 표시
- 사용자가 목록에서 분석 완성도를 빠르게 파악하게 함

## 반응형 지침

### 데스크톱

- 3-4열 에디토리얼 그리드 사용
- 구분선으로 영역을 나누고 카드 그림자 사용은 피함
- 대시보드는 첫 화면에서 핵심 뉴스와 분석 요약이 함께 보여야 함
- 뉴스 목록은 한 화면을 하나의 편집 지면처럼 구성하고 하단 페이지네이션을 제공

### 태블릿

- 2열 구조로 축소
- 필터는 상단 접이식 영역 또는 우측 드로어
- 관련기사와 분석 패널은 본문 아래로 이동 가능

### 모바일

- 단일 컬럼
- 상단 고정 영역은 최소화
- `최근기사 불러오기`, 검색, 필터 버튼을 먼저 제공
- 제목 크기는 줄바꿈을 고려해 과도하게 키우지 않음
- 이미지보다 요약과 키워드를 우선 표시
- 페이지네이션은 이전/다음과 현재 페이지 중심으로 단순화

## 접근성 지침

- 텍스트와 배경 대비는 WCAG AA 이상을 목표로 함
- 하이라이트 색상만으로 의미를 전달하지 않음
- 버튼과 필터는 키보드 포커스 상태를 명확히 제공
- 감정 라벨은 텍스트로 반드시 표시
- 수집/분석 실패 상태는 색상뿐 아니라 문장으로 표시

## 피해야 할 것

- 둥근 카드가 반복되는 일반 SaaS 대시보드 느낌
- 과한 그라데이션, 장식용 블러, 장식용 오브
- 모든 영역을 박스 카드 안에 넣는 구성
- 이미지가 없는 기사에 무의미한 플레이스홀더 박스 사용
- 키워드와 감정 분석을 작은 배지로만 처리해 읽기 어렵게 만드는 것
- 모바일에서 헤드라인이 버튼이나 필터를 밀어내는 구성

## devk 구현 전달 요약

- CSS 변수 또는 디자인 토큰으로 색상, 폰트, 간격을 먼저 정의합니다.
- 기본 배경은 `surface.paper`를 사용하고, 박스 그림자보다 구분선과 여백으로 구조를 만듭니다.
- 헤드라인용 세리프 폰트와 UI용 산세리프 폰트를 분리합니다.
- 뉴스 이미지는 흑백/저채도 처리가 가능하도록 CSS 필터를 둡니다.
- `Article Teaser`, `Keyword Highlight`, `Sentiment Label`, `Analysis Strip`, `Refresh Button` 컴포넌트를 우선 구현합니다.
- 그리드 탐색을 위해 `Editorial Grid`, `Editorial Slot`, `Pagination`, `Source Profile` 컴포넌트를 구현합니다.
- 목록, 상세, 키워드, 감정, 의미 검색, 수집 상태 화면 모두 같은 헤더와 그리드 규칙을 공유합니다.
- 모든 데이터 상태에 대해 `loading`, `empty`, `partial`, `error`, `success` 상태 UI를 준비합니다.
