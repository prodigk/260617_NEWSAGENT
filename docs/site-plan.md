# IT 뉴스 요약 및 분석 사이트 기획

## 목표

이 사이트는 최신 IT 뉴스를 약 50개 단위로 수집하고, 사용자가 빠르게 읽고 판단할 수 있도록 요약, 키워드 분석, 감정 분석, 관련기사 추천을 제공하는 뉴스 분석 웹사이트입니다.

핵심 목표는 단순 뉴스 목록이 아니라 "지금 중요한 기술 이슈가 무엇인지", "각 이슈가 어떤 분위기와 맥락을 갖는지", "더 읽어야 할 관련 기사는 무엇인지"를 한 화면에서 파악하게 하는 것입니다.

## 대상 사용자

- IT 트렌드를 빠르게 파악해야 하는 개발자
- 기술/서비스 기획자
- 스타트업 및 IT 기업 실무자
- AI, 보안, 클라우드, 반도체, 빅테크 동향을 추적하는 사용자
- 여러 뉴스 출처를 매번 직접 확인하기 어려운 사용자

## 핵심 가치

- 여러 IT 뉴스를 한 번에 모아봅니다.
- 긴 기사를 짧고 일관된 형식으로 요약합니다.
- 반복해서 등장하는 핵심 키워드를 파악합니다.
- 기사 분위기와 시장 반응을 감정 분석으로 확인합니다.
- 하나의 기사에서 끝나지 않고 관련 맥락으로 이어지는 기사를 추천합니다.

## 주요 기능 범위

### 1. 최신뉴스 수집

최신 IT 뉴스를 한국어 기사 기준 약 50개까지 수집합니다.

수집 대상은 MVP 단계에서 다음 기준을 따릅니다.

- News API 기반 수집
- 한국어 기사만 노출
- News API가 제공하는 URL을 기준으로 원문 본문 추출까지 수행
- IT, AI, 보안, 클라우드, 반도체, 스타트업, 빅테크 관련 기사
- 최근 발행 기사 우선
- 중복 기사 제거
- 제목, 원문 URL, 기사 이미지, 본문, 출처, 출처 프로필 이미지, 발행일, 요약 가능한 원문 정보 저장

수집은 사이트 최초 실행 시 1회 진행하고, 이후에는 사용자가 "최근기사 불러오기" 버튼을 눌렀을 때 수동으로 갱신합니다.

사용자에게는 수집된 뉴스의 총 개수, 마지막 수집 시간, 수집 성공/실패 상태, 수동 갱신 버튼을 보여줍니다.

수집된 뉴스는 일반 리스트뿐 아니라 첨부 이미지처럼 한 화면에 여러 기사가 편집 배치되는 에디토리얼 그리드 페이지로 보여줍니다. 사용자는 하단 페이지네이션의 `1 2 3 4 5 ...`를 눌러 다음 그리드 페이지로 이동하고, 각 페이지마다 다른 기사 묶음과 다른 크기의 기사 배치를 확인할 수 있습니다.

### 2. 기사 요약

각 뉴스 기사는 일관된 형식으로 요약합니다.

기본 요약 구성은 다음과 같습니다.

- 한 줄 핵심 요약
- 3줄 상세 요약
- 왜 중요한가
- 관련 기업/기술
- 사용자가 더 확인해야 할 포인트

요약 결과는 원문 기사와 연결되어야 하며, 원문 링크를 항상 제공합니다.

### 3. 키워드 분석

수집된 50개 기사 전체와 개별 기사 단위로 키워드를 분석합니다.

분석 항목은 다음과 같습니다.

- 기사별 핵심 키워드
- 전체 기사에서 자주 등장한 키워드
- 급상승 키워드
- 키워드별 관련 기사 목록
- 카테고리별 주요 키워드

사용자는 키워드를 클릭해 관련 기사만 필터링할 수 있어야 합니다.

### 4. 감정 분석

기사의 분위기 또는 시장 반응을 감정 분석으로 표시합니다.

MVP에서는 다음 3단계로 시작합니다.

- 긍정
- 중립
- 부정

추가로 가능하면 감정 점수를 함께 제공합니다.

예시:

- 긍정 72
- 중립 55
- 부정 81

감정 분석은 기사 내용 자체의 논조, 시장 반응, 기업/기술에 대한 평가를 기준으로 합니다. 단, 감정 분석 결과가 사실 판단을 의미하지 않는다는 안내가 필요합니다.

### 5. 관련기사 추천

사용자가 특정 기사를 열었을 때, 비슷한 주제의 관련 기사를 추천합니다.

추천 기준 후보는 다음과 같습니다.

- 공통 키워드
- 같은 기업 또는 기술명
- 같은 카테고리
- 비슷한 감정 분석 결과
- 발행 시점이 가까운 기사

관련기사 추천은 기사 상세 화면 하단 또는 우측 영역에 표시합니다.

MVP의 관련기사 추천은 Vector DB를 사용한 의미 기반 유사도 추천을 기본 방향으로 합니다. 키워드 기반 추천은 추천 사유를 설명하거나 Vector DB 검색 결과를 보조하는 용도로 사용합니다.

### 6. 의미 기반 기사 검색

Vector DB를 활용해 단순 키워드 일치가 아닌 의미 기반 검색을 제공합니다.

검색 예시:

- "AI 규제 관련 뉴스"
- "삼성 반도체 투자 이슈"
- "보안 사고와 개인정보 유출"
- "클라우드 비용 절감"

MVP에서는 검색창 입력값을 기사 임베딩과 비교해 관련도가 높은 기사를 보여주는 방식으로 시작합니다.

### 7. 에디토리얼 그리드 페이지네이션

첨부 이미지의 신문식 그리드 레이아웃을 뉴스 탐색의 주요 화면으로 사용합니다.

기본 방향:

- 수집된 최신뉴스 50개를 페이지 단위로 나눕니다.
- 한 페이지에는 8-12개 기사 정도를 배치합니다.
- 각 페이지는 대표 기사, 중간 기사, 짧은 기사, 추천 기사 영역이 섞인 그리드로 구성합니다.
- 사용자는 하단의 `1 2 3 4 5 ...` 페이지네이션을 눌러 페이지를 이동합니다.
- 페이지를 이동하면 같은 그리드 구조 안에서 기사 내용, 이미지, 키워드, 감정, 출처 정보가 바뀝니다.
- 페이지 이동은 전체 새로고침보다 부드러운 화면 갱신을 우선합니다.

그리드 안의 기사 카드에는 다음 요소를 가능한 범위에서 표시합니다.

- 기사 제목
- 기사 이미지
- 한 줄 요약 또는 본문 일부
- 출처명
- 출처 프로필 이미지 또는 출처 로고
- 발행일
- 핵심 키워드
- 감정 라벨
- 관련기사 수

## 주요 사용자 흐름

### 흐름 1. 최신 IT 이슈 빠르게 파악

1. 사용자가 대시보드에 접속합니다.
2. 최근 수집된 뉴스 50개 중 중요도 높은 기사와 주요 키워드를 확인합니다.
3. 에디토리얼 그리드에서 대표 기사와 보조 기사를 훑습니다.
4. 하단 페이지네이션을 눌러 다음 뉴스 묶음으로 이동합니다.
5. 관심 있는 키워드를 클릭합니다.
6. 해당 키워드와 관련된 기사 목록을 확인합니다.
7. 기사 요약과 감정 분석을 보고 상세 기사를 선택합니다.

### 흐름 2. 특정 기사 맥락 이해

1. 사용자가 뉴스 목록에서 기사를 선택합니다.
2. 기사 상세 화면에서 한 줄 요약, 3줄 요약, 키워드, 감정 분석을 확인합니다.
3. "왜 중요한가"와 "더 확인할 포인트"를 읽습니다.
4. 관련기사 추천 목록에서 같은 이슈의 다른 관점을 확인합니다.

### 흐름 3. 오늘의 기술 트렌드 확인

1. 사용자가 키워드 분석 화면에 접속합니다.
2. 전체 수집 기사에서 가장 많이 등장한 키워드를 확인합니다.
3. 긍정/중립/부정 감정 비율을 함께 봅니다.
4. 특정 키워드의 관련 기사 묶음을 확인합니다.

## 기술 기획

### 기술 구성 방향

MVP는 다음 기술 조합을 전제로 기획합니다.

- News API: 최신 한국어 IT 뉴스 수집
- OpenAI API: 기사 요약, 키워드 분석, 감정 분석, 임베딩 생성
- LangChain: OpenAI 호출을 포함한 기사 요약, 키워드 분석, 감정 분석, 추천 사유 생성 파이프라인 구성
- Chroma: 기사 임베딩 저장, 관련기사 추천, 의미 기반 검색
- Python 웹 애플리케이션: 전체 백엔드와 분석 파이프라인 구현

### 처리 파이프라인

기본 데이터 흐름은 다음과 같습니다.

1. 사용자가 사이트에 처음 접속하거나 "최근기사 불러오기" 버튼을 누릅니다.
2. News API에서 한국어 IT 관련 최신뉴스를 최대 50개 수집합니다.
3. 중복 URL, 유사 제목, 동일 출처 중복 기사를 제거합니다.
4. 수집한 기사 URL에서 원문 본문을 추출합니다.
5. 기사 메타데이터와 추출 본문 또는 요약 가능한 텍스트를 저장합니다.
6. LangChain 파이프라인과 OpenAI API로 기사 요약, 키워드 분석, 감정 분석을 실행합니다.
7. 기사 제목, 본문, 요약, 키워드를 합친 텍스트로 OpenAI 임베딩을 생성합니다.
8. 임베딩을 Chroma에 저장합니다.
9. Chroma 유사도 검색으로 관련기사를 계산합니다.
10. 기사 중요도, 이미지 보유 여부, 키워드, 감정, 발행 시간을 기준으로 에디토리얼 그리드 페이지를 구성합니다.
11. 목록, 상세, 키워드 분석, 감정 분석, 관련기사 추천 화면에 결과를 표시합니다.

### LangChain 사용 범위

LangChain은 LLM 호출을 직접 화면에서 수행하는 도구가 아니라, 백엔드 분석 파이프라인을 일관되게 관리하는 계층으로 사용합니다.

MVP에서 LangChain이 담당할 범위는 다음과 같습니다.

- 기사 요약 체인
- 키워드 추출 체인
- 감정 분석 체인
- 추천 사유 생성 체인
- OpenAI 모델 호출 관리
- 기사 분석 결과를 정해진 JSON 형태로 반환하도록 구조화

LangChain 출력은 화면에 바로 표시하기 전에 데이터베이스에 저장합니다.

### OpenAI 사용 범위

OpenAI API는 기사 분석과 임베딩 생성을 위해 사용합니다.

MVP에서 OpenAI가 담당할 범위는 다음과 같습니다.

- 한 줄 요약 생성
- 3줄 요약 생성
- "왜 중요한가" 분석 생성
- 핵심 키워드 추출
- 긍정/중립/부정 감정 분석
- 추천 사유 생성
- 기사 의미 검색과 관련기사 추천을 위한 임베딩 생성

OpenAI API 키는 환경 변수로 관리하며, 키가 없거나 잘못된 경우 사용자가 이해할 수 있는 설정 오류를 표시합니다.

### Vector DB 사용 범위

Vector DB는 관련기사 추천과 의미 검색을 위해 사용하며, MVP에서는 Chroma를 사용합니다.

MVP에서 Vector DB가 담당할 범위는 다음과 같습니다.

- 기사별 임베딩 저장
- 특정 기사와 유사한 기사 검색
- 자연어 검색어와 관련도 높은 기사 검색
- 관련기사 추천 점수 계산 보조

Chroma는 로컬 개발과 MVP 검증이 쉬운 Vector DB로 사용하며, 추후 Qdrant 같은 서버형 Vector DB로 교체할 수 있도록 저장 계층을 분리합니다.

### News API 사용 범위

News API는 최신 기사 수집의 1차 데이터 소스로 사용합니다.

요구사항:

- 한국어 기사 검색 가능
- IT 관련 검색어 또는 카테고리 기반 수집 가능
- 제목, URL, 출처, 발행일 제공
- 가능하면 요약 또는 본문 일부 제공
- API 실패, 호출 제한, 빈 결과에 대한 상태 처리 필요

News API가 본문 전체를 제공하지 못하는 경우, 기사 URL을 이용해 원문 본문 추출을 수행합니다.

수집 검색어 후보:

- IT
- 인공지능
- AI
- 보안
- 클라우드
- 반도체
- 스타트업
- 빅테크
- 소프트웨어
- 데이터센터

### 분석 결과 저장 원칙

- 원문 기사 데이터와 LLM 분석 결과를 분리해 저장합니다.
- LangChain과 OpenAI 실행 결과는 재생성 가능하도록 사용 모델, 생성 시간, 입력 텍스트 범위를 함께 기록합니다.
- Chroma에 저장한 임베딩은 원문 DB의 기사 ID와 반드시 연결합니다.
- 같은 기사를 다시 수집해도 기존 분석 결과를 재사용할 수 있어야 합니다.

### 실패 처리 요구사항

- News API 호출 실패 시 기존에 저장된 기사 목록을 계속 보여줍니다.
- 원문 URL 본문 추출 실패 시 News API에서 받은 제목과 설명 텍스트만으로 분석을 시도하되, 본문 추출 실패 상태를 기록합니다.
- 일부 기사 분석이 실패해도 전체 수집 작업은 중단하지 않습니다.
- LangChain 분석 실패 기사는 "분석 대기" 또는 "분석 실패" 상태로 표시합니다.
- OpenAI API 호출 실패 시 해당 기사는 "분석 실패" 상태로 표시합니다.
- Chroma 저장 실패 시 관련기사 추천과 의미 검색만 제한하고 기본 뉴스 목록은 유지합니다.
- API 키가 없거나 잘못된 경우 설정 오류를 명확히 보여줍니다.

## MVP 우선순위

### 1순위

- 최신뉴스 50개 수집
- 에디토리얼 그리드 기반 뉴스 목록 화면
- 페이지네이션
- 기사 상세 화면
- 기사 요약
- 기사별 키워드 표시
- 기사별 감정 분석 표시
- Vector DB 기반 관련기사 추천

### 2순위

- 키워드 분석 화면
- 감정 분석 대시보드
- 필터와 검색
- 수집 상태 화면
- 의미 기반 기사 검색

### 3순위

- 사용자 관심 키워드 저장
- 매일 요약 리포트
- 알림 기능
- 출처별 신뢰도 표시
- 키워드 변화 추이 그래프

## 기획상 주요 결정

- 최신뉴스 수집 단위는 우선 50개로 설정합니다.
- 수집된 뉴스는 기본적으로 에디토리얼 그리드 페이지로 탐색합니다.
- 에디토리얼 그리드는 페이지네이션을 통해 여러 페이지로 나누어 보여줍니다.
- 뉴스 수집 방식은 News API 기반으로 설정합니다.
- News API로 수집한 기사 URL에서 원문 본문 추출까지 수행합니다.
- 뉴스 언어는 한국어 중심으로 설정합니다.
- 뉴스 갱신은 사이트 최초 실행 시 News API 키가 있으면 초기 데이터를 자동 수집하고, 이후에는 "최근기사 불러오기" 버튼으로 수동 갱신합니다.
- News API 키가 없거나 초기 수집에 실패하면 화면 확인을 위해 샘플 데이터로 대체하되, 수집 상태 화면에 fallback 상태를 표시합니다.
- 기사 요약, 키워드 분석, 감정 분석은 LangChain과 OpenAI API 기반 분석 파이프라인으로 구성합니다.
- 관련기사 추천과 의미 기반 검색은 Chroma를 활용합니다.
- MVP는 로그인 없이 사용할 수 있는 단일 사용자 웹사이트로 가정합니다.
- 감정 분석은 긍정/중립/부정 3분류로 시작합니다.
- 추천은 Chroma 기반 유사도 추천을 기본으로 하고, 공통 키워드는 추천 사유 설명에 활용합니다.
- 첫 화면은 대시보드로 구성합니다.

## 확인이 필요한 사항

다음 항목은 구현 전에 사용자의 확인이 필요합니다.

1. 감정 분석은 기사 논조 기준으로 볼까요, 시장/투자 반응 기준으로 볼까요?

2. MVP에서 로그인 기능이 필요한가요?

3. 사용할 OpenAI 모델과 임베딩 모델은 `devk`가 비용과 성능 기준으로 선택해도 될까요?

## designk에게 전달할 내용

- 첫 화면은 대시보드입니다.
- 정보 우선순위는 수집 상태, 최근기사 불러오기 버튼, 핵심 뉴스, 상위 키워드, 감정 분포, 추천 이슈 순서입니다.
- 뉴스 목록의 기본 레이아웃은 첨부 이미지의 신문식 그리드이며, 하단 페이지네이션으로 그리드 페이지를 전환합니다.
- 각 그리드 페이지는 대표 기사, 중간 기사, 짧은 기사, 분석/추천 영역이 섞이도록 설계합니다.
- 뉴스 상세 화면은 요약과 분석을 원문보다 먼저 보여줘야 합니다.
- 키워드와 감정은 필터로 바로 이어질 수 있는 인터랙션 요소가 되어야 합니다.
- 전체 톤은 첨부 이미지 기반의 에디토리얼 뉴스 매거진 감성과 정보 밀도 높은 IT 분석 도구의 사용성을 결합합니다.
- 구체적인 디자인 파운데이션과 화면 구조는 `docs/design-guidelines.md`를 기준으로 합니다.

## devk에게 전달할 내용

- Python 기반 웹사이트로 구현합니다.
- 최신뉴스 50개 수집을 기본 배치 단위로 설계합니다.
- 뉴스 수집은 News API 기반이며, 한국어 중심입니다.
- News API 수집 결과의 URL을 이용해 원문 본문 추출까지 수행합니다.
- 최초 실행 시 1회 수집하고, 이후에는 사용자의 "최근기사 불러오기" 버튼 액션으로 수동 갱신합니다.
- 첫 실행 초기 화면은 News API 키가 있을 때 News API 기사 데이터로 부트스트랩합니다.
- 초기 부트스트랩은 화면을 빠르게 노출하기 위해 OpenAI 호출 없이 휴리스틱 요약/키워드/감정 분석을 먼저 적용하고, 수동 갱신에서 전체 분석 파이프라인을 실행합니다.
- 원문 기사, 요약, 키워드, 감정 분석, 추천 결과는 분리된 데이터 모델로 저장합니다.
- 기사 표시를 위해 제목, 기사 이미지, 본문/설명, 출처명, 출처 URL, 출처 프로필 이미지 또는 로고, 발행일, 작성자명, 작성자 프로필 이미지가 가능하면 저장되어야 합니다.
- 기사 이미지나 출처 프로필 이미지가 없을 경우 대체 UI를 제공할 수 있도록 필드 누락 상태를 보존합니다.
- 50개 기사를 페이지 단위로 나누고, 각 페이지에 들어갈 기사와 슬롯 타입을 계산하는 에디토리얼 그리드 구성 로직을 구현합니다.
- 요약, 키워드 분석, 감정 분석은 LangChain과 OpenAI API 파이프라인으로 구성합니다.
- 기사 임베딩은 Chroma에 저장하고, 관련기사 추천과 의미 검색에 사용합니다.
- News API, OpenAI 모델, Embedding 모델, Chroma 저장 계층은 나중에 교체 가능하도록 모듈화합니다.
- 초기 추천은 Chroma 유사도 검색을 기본으로 하고, 공통 키워드는 추천 이유 설명에 활용합니다.
- 화면 구현 시 `docs/design-guidelines.md`와 `docs/design-dev-handoff.md`를 함께 참고합니다.
- 핵심 UI 컴포넌트는 `AppHeader`, `RefreshNewsButton`, `ArticleTeaser`, `KeywordHighlight`, `SentimentLabel`, `AnalysisStrip`, `RelatedArticleList`, `SemanticSearchBox`, `JobTimeline`을 우선 구현합니다.
- 에디토리얼 그리드를 위해 `EditorialGrid`, `EditorialSlot`, `Pagination`, `SourceProfile` 컴포넌트를 추가로 구현합니다.

## qak에게 전달할 내용

- 최초 실행 시 뉴스 수집이 1회 수행되는지 검증해야 합니다.
- "최근기사 불러오기" 버튼을 눌렀을 때만 수동 갱신되는지 검증해야 합니다.
- 수집 기사 수가 50개 미만일 때의 화면과 상태를 검증해야 합니다.
- 뉴스 목록에서 페이지네이션을 눌렀을 때 해당 페이지의 기사 묶음으로 에디토리얼 그리드가 변경되는지 검증해야 합니다.
- 마지막 페이지에 8-12개보다 적은 기사가 있어도 빈 슬롯 없이 자연스럽게 배치되는지 검증해야 합니다.
- 필터/검색 적용 후 페이지네이션 총 페이지 수와 현재 페이지가 올바르게 갱신되는지 검증해야 합니다.
- 중복 기사 제거가 동작하는지 확인해야 합니다.
- 기사 이미지, 출처 프로필 이미지, 작성자 정보가 누락된 기사에서도 대체 UI가 정상적으로 표시되는지 확인해야 합니다.
- 원문 URL 본문 추출 성공/실패 상태가 기록되는지 확인해야 합니다.
- 요약, 키워드, 감정 분석, 추천 결과가 원문 기사와 정확히 연결되는지 확인해야 합니다.
- 분석 실패 또는 외부 API 실패 시 사용자에게 상태가 표시되어야 합니다.
- News API 실패 시 기존 기사 목록이 유지되는지 검증해야 합니다.
- LangChain 분석 실패 시 해당 기사만 실패 상태로 표시되는지 검증해야 합니다.
- OpenAI API 실패 시 해당 기사만 분석 실패 상태로 표시되는지 검증해야 합니다.
- Chroma 저장 또는 검색 실패 시 관련기사 추천과 의미 검색만 제한되는지 검증해야 합니다.
- 감정 분석 결과가 비어 있거나 애매한 기사에서도 화면이 깨지지 않아야 합니다.
